# Plugin UX Flow

## Цель

Этот документ фиксирует основной пользовательский сценарий от установки plugin до создания или подключения к существующему sync vault и базовой настройки.

---

## Основная user story

Пользователь хочет установить Obsidian plugin, подключить его к sync-серверу, создать новый vault или присоединиться к существующему, настроить синхронизацию и убедиться, что всё работает.

---

## Основной сценарий

1. Пользователь устанавливает plugin в `.obsidian/plugins/obsidian-sync-plugin/`.
2. Включает plugin в `Community plugins`.
3. Открывает настройки plugin.
4. В секции `Connection` вводит:
   - `Server URL`
   - `Auth token`, если сервер его требует
   - при необходимости проверяет `Device ID`
5. Нажимает `Check` рядом с `Server URL`.
6. Получает один из результатов:
   - подключение успешно
   - ошибка авторизации
   - сервер недоступен
7. После успешной проверки переходит к секции `Vault`.
8. Plugin автоматически показывает список vault с сервера, если сервер доступен.
9. Если нужно, пользователь может вручную обновить список через `Load vaults`.

---

## Ветка Create

1. Пользователь не находит нужный vault в уже показанном server registry.
2. В `Vault flow` вводит новый `vaultId`.
3. Нажимает `Create & join`.
4. Plugin:
   - создаёт vault на сервере
   - переключает текущий клиент на этот `vaultId`
   - сохраняет локальный state и scope для этого vault
5. Пользователь видит, что новый vault стал активным.

---

## Ветка Join

1. Пользователь сразу видит нужный vault в списке `Available vaults`.
2. Нажимает `Join` или выбирает его через `Join server vault`.
3. Plugin переключается на этот `vaultId`.
4. Пользователь видит новый `Current vault`.

---

## Дальнейшая настройка

После выбора vault пользователь при необходимости настраивает:

- `Auto sync`
- `Poll interval`
- `Include patterns`
- `Ignore patterns`
- `E2EE passphrase`

Дополнительно пользователь может:

- открыть `Devices` и проверить registry устройств
- посмотреть `Sync health`
- проверить effective sync scope

---

## Первый успешный outcome

1. Пользователь запускает `Sync now` или ждёт auto sync / realtime push.
2. Видит, что файл загрузился на сервер или подтянулся с сервера.
3. В status bar и `Overview` нет ошибки.
4. Текущий статус становится `Up to date`.

---

## Ментальная модель пользователя

Пользователь должен понимать разделение ответственности:

- `Connection` отвечает за доступ к серверу
- `Vault` отвечает за выбор sync-пространства
- `Sync Scope` определяет, какие файлы участвуют в sync
- `E2EE` шифрует содержимое файла
- `Devices` и `Sync health` нужны для диагностики

---

## Короткий happy path

1. Установить plugin.
2. Ввести `Server URL` и `Auth token`.
3. Нажать `Check`.
4. Выбрать `Create & join` или `Join` из уже загруженного списка vault.
5. При необходимости нажать `Load vaults`, чтобы обновить список.
6. Нажать `Sync now`.
7. Убедиться, что статус `Up to date`.
