## Изначальная цель

Первоначальная задача была такой:

- локально редактировать данные в Obsidian vault
- синхронизировать изменения через простой сервер
- сохранять конфликты явно, а не пытаться их автоматически merge-ить
- держать архитектуру достаточно простой для MVP

Эта цель достигнута, после чего проект был расширен.

---

## Архитектура в текущем виде

### Сервер

Сервер на Rust:

- принимает upload/delete/restore
- хранит текущую версию файлов
- ведёт change feed
- хранит историю версий
- ведёт реестр vault и устройств
- отдаёт realtime-сигналы через SSE

### Plugin

Obsidian plugin:

- хранит локальное sync state в plugin data
- сканирует локальный vault
- применяет include/ignore scope
- загружает локальные изменения
- догоняет сервер по `changes`
- подписывается на `events`
- поддерживает content-only E2EE
- показывает history/restore UI

---

## Что осталось от исходного MVP-дизайна

Эти идеи по-прежнему актуальны:

- whole-file sync
- `base_version` как основа conflict detection
- conflict copy вместо auto-merge
- `rename` как `delete + create`
- приоритет корректности данных над оптимизацией
